saya akan membuat aplikasi tauri v2 dengan node js dan express js 
- tauri v2 sebagai frontend 
- node js dan express js sebagai backend 
- sqlite3 sebagai database

fokus ke desain user interface yang modern dan responsif menggunakan native js tanpa framework

struktur folder sebagai berikut:
- server
    berisi kode server node js dan express js 
- src
    berisi kode frontend menggunakan tauri v2 dan native js tanpa framework

menu tambah kontak, import dan sync whatsapp letakan di sebelah kanan form pencarian begitu juga di tags

di modal tambah kontak, fungsi tags menampilkan data dari tag seperti radio yang bisa di pilih,
model tampilan seperti badge yang bisa di klik untuk mengaktifkan dan menonaktifkan nya, jika dipilih warna nya menjadi cerah hijau active dan jika dinonaktifkan warna nya menjadi abu abu, text nya putih 

logika di databse 1 tags bisa beberapa kontak, dan 1 kontak bisa memiliki beberapa tags. 
kemudian setiap user memiliki kontak dan tags nya sendiri, jadi kontak dan tags tidak bisa di share antar user begitupun template pesan juga.

owh saya lupa, seharusnya di database kontak ada kolom ferifikasi nomor, apakah nomor tersebut sudah terdaftar di whatsaap atau belum, fungsi ini sangat bagus agar memfilter nomor, kemudian di tabel list kontak tambahkan th ini, dengan tampilan data nantinya terdaftar atau tidak dengan badge, jika terdaftar badge su


di tab kontak depan tombol tambah kontak, tambahkan tombol verifikasi, yang fungsinya akan memverifikasi nomor whatsapp yang ada di database, apakah nomor tersebut sudah terdaftar di whatsaap atau belum,
dengan menampilkan badge apakah nomor tersebut terdaftar atau tidak, warna badge terdaftar hijau dan tidak terdaftar merah, belum verifikasi abu abu. 

jadi logika di data base untuk verifikasi nomor adalah : 
status_verifikasi : 
0 = belum verifikasi, 1 = terverifikasi, 2 = gagal verifikasi 

kemudian tambahkan tabel settings di database yang digunakan untuk menyimpan pengaturan :
- delay pengiriman pesan blast,satuan nya detik milliseconds. 
- delay verifikasi nomor, satuan nya detik milliseconds. 
- random delay pengiriman pesan blast, satuan nya detik milliseconds. 
- random delay verifikasi nomor, satuan nya detik milliseconds. 

fungsi random ini gunanya untuk mengirim pesan blast secara acak, jadi pesan blast tidak akan terkirim secara berurutan, ini untuk menghindari terdeteksi spam.

kemudian masukkan ke database tabel settings : blast_delay, verification_delay, random_blast_delay, random_verification_delay, 
default nya adalah : blast_delay = 7000, verification_delay = 5000, random_blast_delay = 2000, random_verification_delay = 2000
data ini akan selalu ada karena defalut, jadi saat database baru dibuat data ini sudah ada, dan setiap user memiliki data ini sendiri, jadi data ini tidak bisa di share antar user.

Sekarang kita fokus dalam pengembangan page blash.

di page blash ada 2 tab, yaitu kirim balst, dan template.

di kirim blast : 
- form 1 : Kirim ke 
  -  jika # akan menampilkan sugesti tag, 
  - jika @ akan menampilkan sugesti nama kontak.
  - jika mengetik nomor langsung misal 08124xxxx maka tidak menampilkan sugesti.
  -format penulisan, #tags,@nama,nomor.

di form 2 : Pesan text
  - from log text dengan maksimal 3000 kata.
  - di atas text area tambahkan emotkon, tambahkan template.

kemudian di bawahnya ada tombol kirim blast.

Logika tombol kirim blast: 
- Tombol terdisable jika form 1 dan form 2 kosong.
- Tombol terdisable jika sedang dalam proses kirim blast.
- Tombol terdisable jika sedang dalam proses reconecting.
- Tombol terdisable jika sedang dalam proses verifikasi nomor.
- Tombol enable jika form 1 dan form 2 terisi.
- Tombol enable jika tidak sedang dalam proses kirim blast.
- Tombol enable jika tidak sedang dalam proses reconecting.
- Tombol enable jika tidak sedang dalam proses verifikasi nomor.

tab template 
    - tombol tambah template  
    - table template 
        - kolom nama 
        - kolom pesan
        - kolom aksi

di kolom aksi :
    - tombol gunakan template.
    - tombol edit template 
    - tombol hapus template 
   
seharusnya tabel blast_history menyimpan data :
    - nama jika nomor sudah tersimpan di database kontak, jika tidak simpan unknown.
    - nomor jika sudah tersimpan di database kontak, jika tidak simpan nomor saja.
    - pesan yang dikirim
    - status  'failed', 'success'
    - user_id
    - created_at
    - updated_at 


Sekarang kita fokus ke page settings, di page settings ada 3 card :

card 1 : pengaturan blast
  - blast_delay
  - blast_random_delay

card 2 : pengaturan verifikasi nomor
  - verification_delay
  - verification_random_delay

card 3 : Cek update software.
    - tombol cek update 
    - menampilkan info software (nama, versi, status update)
    - status update : "UP-TO-DATE" atau "UPDATE AVAILABLE"

    saat tombol cek update di klik:
        - icon berubah menjadi spiner
        - disabled tombol cek update
        - menampilkan info proses "Checking for updates..."

        saat selesai:
            - icon kembali menjadi semula
            - enabled tombol cek update
            - menampilkan info update :
                - versi saat ini
                - versi baru
                - status "UP-TO-DATE" atau "UPDATE AVAILABLE"
                - tombol download update
                - tombol install update (jika ada update)

logika update mengikuti tauri updater dan di update diambil rari repository github.com/candrasp/KenWa/releases
